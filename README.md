计划做一个口腔医院医务系统，需要支持以下功能
1. 支持医患管理，医患前期咨询、挂号、医疗进展/计划，支持对接到微信公众号与小程序
2. 支持医院内工单管理
3. 支持医院耗材管理
4. 支持医院内员工IM交流
5. 需要支持windows网页/客户端、android/ios
6. 需要支持多个医院，各医院间信息隔离
7. 需要支持所有人员的主页、博客和论坛系统
8. 支持口腔医院相关商城系统，支持医院管理者和采购者采购相关物品
9. 后续计划支持宠物医院，系统需要支持扩展
10. 支持医务人员与管理者查看业绩报表信息
11. 计划使用AI辅助开发，系统本身不使用AI，考虑架构支持后续引入AI。
12. 系统软件选型考虑到整个系统开发及后续维护工作量，可集成或基于开源软件开发
13. 架构设计需要考虑方便快速扩展支持高性能、高并发
14. 系统计划对外免费提供，但通过公众号/小程序/app系统推送口腔健康相关广告、手机端IM系统增加医护和患者使用时间、商城系统收取广告排名费用等方式盈利。
